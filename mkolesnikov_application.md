**[ НАЗВАНИЕ ПРОЕКТА ]**

"Learning Copilot – ИИ-ассистент для работы с учебными материалами"

**[ ПРОБЛЕМНОЕ ПОЛЕ ]**

При самостоятельном обучении студенты и школьники сталкиваются с рядом трудностей: поиск нужной информации в больших объёмах материалов занимает много времени, сложно проверить себя без готовых тестов, а регулярное повторение требует дополнительных усилий на подготовку вопросов. Существующие решения (ChatGPT, Claude, Notion AI) работают с общими знаниями или отдельными документами, но не предоставляют комплексного инструмента для работы именно с личной библиотекой учебных материалов. Кроме того, в таких решениях отсутствует возможность генерировать тесты строго по загруженным источникам и отслеживать результаты. Заявляемый продукт решает эти проблемы, предоставляя единую систему для работы с конспектами: объяснения по темам с цитатами, генерация тестов по материалу и отслеживание результатов.

**[ ЗАКАЗЧИК / ПОТЕНЦИАЛЬНАЯ АУДИТОРИЯ ]**

Целевая аудитория продукта:
* Студенты вузов, готовящиеся к экзаменам и работающие с большими объёмами конспектов
* Школьники старших классов, которым нужна помощь в подготовке к ОГЭ/ЕГЭ
* Самообучающиеся специалисты (программисты, изучающие новые технологии, специалисты в других областях)
* Преподаватели и репетиторы, которым нужно быстро готовить проверочные вопросы по материалам

**[ АППАРАТНЫЕ / ПРОГРАММНЫЕ ТРЕБОВАНИЯ ]** 

Продукт разрабатывается как веб-приложение, доступное через браузер:
* Браузеры: Chrome (версия 90+), Firefox (версия 88+), Safari (версия 14+), Edge (версия 90+)
* Интернет-соединение для работы с LLM API
* Рекомендуемое разрешение экрана: от 1280×720(адаптив под другие устройства предусмотрен, это просто рекомендация)
* Серверные требования: PostgreSQL 14+, Python 3.11+

**[ ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ ]**

Программный продукт будет предоставлять следующие возможности:
* Регистрация и авторизация пользователей
* Загрузка учебных материалов (текстовые файлы, Markdown, PDF с текстовым слоем)
* Индексация материалов для быстрого поиска по содержанию
* RAG-тьютор: ответы на вопросы по загруженным материалам с обязательными цитатами и ссылками на источники или обьяснения темы на основе общих знаний модели.
* RAG-квизер: генерация тестов (множественный выбор, короткий ответ, верно/неверно, развёрнутый ответ) строго по содержанию источников, также возможен режим прямого диалога в процессе которого агент будет подчёркивать неточности и помогать с трудностями
* Прохождение тестов с автоматической проверкой и объяснением правильных ответов
* История диалогов с тьютором и сохранённые наборы тестов
* Статистика прогресса: количество пройденных тестов, процент правильных ответов, проблемные темы

**[ ПОХОЖИЕ / АНАЛОГИЧНЫЕ ПРОДУКТЫ ]**

Анализ существующих решений показал следующее:

* ChatGPT / Claude: работают с общими знаниями, не сохраняют личную библиотеку материалов, не генерируют структурированные тесты с привязкой к конкретным фрагментам источников
* Notion AI: интегрирован только с Notion, нет специализированных функций для обучения (генерация тестов, отслеживание прогресса)
* ChatPDF / AskYourPDF: работают с отдельными документами, не предоставляют комплексного решения (тьютор + квизинг + статистика), отсутствует возможность работы с несколькими источниками одновременно
* Quizlet: требует ручного создания карточек и тестов, не работает автоматически по загруженным материалам

**[ ИНСТРУМЕНТЫ РАЗРАБОТКИ, ИНФОРМАЦИЯ О БД ]**

* Python / FastAPI – backend и REST API
* LangGraph и(или) другие фреймворки LLM – оркестрация RAG-агентов (тьютор и квизер)
* PostgreSQL – хранение пользователей, материалов, истории
* pgvector или chroma – векторное хранилище для поиска по содержанию
* React / Next.js – frontend (личный кабинет, чат, тесты)
* Docker – контейнеризация для развёртывания
